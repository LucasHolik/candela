# Plan: Black & White Filter Toggle

## Context

Candela currently lets the user adjust per-monitor software brightness, hardware brightness, and colour temperature, all routed through a single `ApplyGammaRamp()` per monitor. The user wants an additional **black & white (grayscale) filter** that behaves like the existing brightness sliders: a toggle in the tray popup (left-click) with corresponding visibility controls in the Settings dialog.

Two operating modes are required:

1. **Per-monitor mode** *(default)* — Each monitor can independently show a B&W toggle in the popup; each toggle controls only its own monitor.
2. **Global mode** — A single B&W toggle appears in the popup affecting all monitors simultaneously. While global mode is on, the per-monitor visibility checkboxes in Settings are disabled and forced off (so no per-monitor toggles render in the popup).

The Settings dialog only controls *what toggles appear* in the popup — it never directly turns the filter on/off. Actual filter state lives in the popup and persists across restarts (matching colour-temp behaviour).

Design philosophy is professional, modular, and extensible — implemented as a clean addition to the existing gamma pipeline using established patterns (per-monitor `MonitorSettings`, control-ID stride scheme, `BrightnessController` static API).

## Design Overview

### Filter pipeline

B&W is applied as a final desaturation step inside the gamma-ramp builder. After `ApplyGammaRamp` composes brightness × colour-temp into the 3×256 ramp, if the filter is active for that monitor, each entry's R/G/B is collapsed to a single luminance value using ITU-R BT.601 weights (`0.299 R + 0.587 G + 0.114 B`). The filter therefore composes naturally with brightness and colour-temp without a separate code path.

### State model

- **`MonitorSettings.showBWToggleInPopup` (bool)** — per-monitor: whether to show the B&W toggle in the tray popup for this monitor. Set from Settings dialog. Disabled while global mode is on.
- **`MonitorSettings.bwFilterEnabled` (bool)** — per-monitor: the actual filter state. Toggled from popup. Persisted and restored on startup.
- **`AppSettings.globalBWMode` (bool)** — global: whether the popup shows a single global toggle instead of per-monitor toggles. Set from Settings dialog.
- **`AppSettings.globalBWEnabled` (bool)** — global: state of the global toggle. Toggled from popup when in global mode. Persisted.

When global mode is on, popup shows one "B&W (All Monitors)" checkbox at the top; per-monitor toggles are hidden. When `globalBWEnabled` flips, `BrightnessController` applies B&W to every monitor in one pass. Per-monitor `bwFilterEnabled` is preserved underneath, so when the user disables global mode, individual selections return.

## Files to Modify

### 1. `src/settings.h` / `src/settings.cpp` — Persistence

**`src/settings.h`:**
- In `struct MonitorSettings` add:
  - `bool showBWToggleInPopup = false;`
  - `bool bwFilterEnabled = false;`
- Add two app-level fields (alongside `m_startOnBoot`):
  - `bool m_globalBWMode = false;`
  - `bool m_globalBWEnabled = false;`
  - Plus public getters/setters mirroring `getStartOnBoot()` / `setStartOnBoot()`.

**`src/settings.cpp`:**
- Add registry value names: `L"ShowBWToggle"`, `L"BWFilterEnabled"` (per-monitor subkey); `L"GlobalBWMode"`, `L"GlobalBWEnabled"` (root key).
- Extend `load()` (around lines 123–149 for monitor subkeys; around line 80 for root) to read the four new DWORDs.
- Extend `save()` (around lines 193–219 monitor; root section) to write them.

### 2. `src/brightness.h` / `src/brightness.cpp` — Filter state & application

**`src/brightness.h`:**
- Add `bool bwFilterEnabled = false;` to `struct Monitor`.
- Add public static API to `BrightnessController`:
  - `static bool SetBWFilter(int monitorIndex, bool enabled);`
  - `static bool GetBWFilter(int monitorIndex);`
  - `static bool SetGlobalBWFilter(bool enabled);` — applies to all monitors in one call.

**`src/brightness.cpp`:**
- Implement the three methods. `SetBWFilter` updates `g_monitors[i].bwFilterEnabled` and calls `ApplyGammaRamp` with the monitor's current brightness/temp + new BW flag. `SetGlobalBWFilter` iterates all monitors and calls `ApplyGammaRamp` with the global flag forced on (without mutating individual `bwFilterEnabled` so per-monitor state is preserved).
- Update existing call sites in `SetSoftwareBrightness` (line 124) and `SetSoftwareColorTemp` (line 138) to pass `monitor.bwFilterEnabled || g_globalBWActive` to `ApplyGammaRamp`. Track `g_globalBWActive` as a file-static bool in `brightness.cpp`, set via `SetGlobalBWFilter`.

### 3. `src/colortemp.h` / `src/colortemp.cpp` — Gamma ramp grayscale step

**`src/colortemp.h`:**
- Change signature: `bool ApplyGammaRamp(HDC hdc, int brightness, int kelvin, bool grayscale);`

**`src/colortemp.cpp`:**
- Inside the ramp-build loop (around lines 50–58), after computing `r * base`, `g * base`, `b * base`, if `grayscale` is true compute `gray = 0.299*R + 0.587*G + 0.114*B` and assign to all three channels.

### 4. `src/gui.cpp` / `src/gui.h` — Tray popup UI

**Add to `GuiConstants` (around lines 24–30):**
- `OFFSET_BW_TOGGLE = 8` (per-monitor popup checkbox).
- A separate well-known ID for the global popup checkbox, e.g. `ID_GLOBAL_BW_TOGGLE = 1900` (outside the per-monitor stride range).

**In `ShowBrightnessSlider()` (lines 85–292):**
- Before the per-monitor loop, if `g_settings.getGlobalBWMode()` is true, create one "B&W (All Monitors)" checkbox at the top of the popup with ID `ID_GLOBAL_BW_TOGGLE`, initialised from `g_settings.getGlobalBWEnabled()`. Skip per-monitor B&W toggles entirely in this mode.
- Otherwise, inside the per-monitor loop after the hardware slider block (line 284), if `settings.showBWToggleInPopup`, add a "B&W" checkbox with ID `baseID + OFFSET_BW_TOGGLE`, initialised from `BrightnessController::GetBWFilter(i)`.
- Adjust the dynamic window-width / height calculation (lines 127–140) to account for the extra row when toggles are visible.

**In `BrightnessSliderProc()` (lines 294–383):**
- Add `WM_COMMAND` / `BN_CLICKED` handling:
  - If control ID == `ID_GLOBAL_BW_TOGGLE`: read checkbox state, call `BrightnessController::SetGlobalBWFilter(state)`, persist via `g_settings.setGlobalBWEnabled(state)` + `g_settings.save()`.
  - Else if ID maps to `OFFSET_BW_TOGGLE`: decode monitor index, read state, call `BrightnessController::SetBWFilter(idx, state)`, update `MonitorSettings.bwFilterEnabled`, persist.

### 5. `src/gui.cpp` — Settings dialog UI

**In `ShowSettingsDialog()` (lines 389–522):**

- In the per-monitor loop, after the hardware checkbox (line 486), add a "Show B&W toggle" checkbox with ID `baseID + OFFSET_SETTINGS_BW_CHECK`, initial state from `settings.showBWToggleInPopup`. Disable it (`EnableWindow(hwnd, FALSE)`) when global mode is on.
- Bump `perMonitorHeight` (line 425) to accommodate the new row.

**In `SettingsProc()` (lines 524–631):**
- Handle `BN_CLICKED` for the global-mode checkbox: update `g_settings.setGlobalBWMode(state)`, persist, then walk all per-monitor "Show B&W toggle" checkboxes calling `EnableWindow(..., !state)` and `SendMessage(BM_SETCHECK, BST_UNCHECKED)` when enabling global mode (and reflect this in `MonitorSettings.showBWToggleInPopup = false`).
- Handle `BN_CLICKED` for the per-monitor "Show B&W toggle" checkbox (`OFFSET_SETTINGS_BW_CHECK`): update `settings.showBWToggleInPopup`, persist. No validation needed (unlike brightness, which requires at least one slider visible).

### 6. `src/main.cpp` — Startup restore

**In `RestoreBrightnessOnStartup()` (lines 147–166):**
- After existing restore calls, if `g_settings.getGlobalBWMode() && g_settings.getGlobalBWEnabled()`, call `BrightnessController::SetGlobalBWFilter(true)`.
- Otherwise, for each monitor with `settings.bwFilterEnabled`, call `BrightnessController::SetBWFilter(i, true)`.

## Reused Patterns

- **Per-monitor stride for control IDs** — `src/gui.cpp` lines 19–53. New offsets `OFFSET_BW_TOGGLE` and `OFFSET_SETTINGS_BW_CHECK` slot into the existing scheme.
- **`MonitorSettings` get/set + `g_settings.save()`** — `src/settings.h` lines 31–36 and existing popup/settings handlers.
- **`ApplyGammaRamp` as the single gamma pipeline** — `src/colortemp.cpp` lines 40–60. All brightness/temp/B&W effects share this entry point.
- **`BrightnessController` static API** — `src/brightness.h` lines 78–116. New `Set/GetBWFilter` and `SetGlobalBWFilter` methods follow the same pattern as `SetSoftwareBrightness` / `SetSoftwareColorTemp`.
- **Checkbox creation + `BM_SETCHECK` / `BN_CLICKED`** — `src/gui.cpp` lines 471–486 (Settings) and the "Start on boot" checkbox (template for the global checkbox in Settings).

## Verification

1. **Build:** `make` from project root produces `build/candela.exe` with no warnings introduced.
2. **Persistence:** Toggle B&W (per-monitor and global), close & relaunch app, confirm state restored. Inspect `HKCU\Software\Candela` and `Software\Candela\Monitors\<device>` in `regedit` for the four new DWORDs.
3. **Per-monitor mode (default):**
   - Settings → tick "Show B&W toggle" for Monitor 1 only → left-click tray → only Monitor 1 row shows a B&W checkbox.
   - Toggle Monitor 1's B&W → confirm only Monitor 1 goes grayscale; brightness and colour-temp continue to work on top.
4. **Global mode:**
   - Settings → enable "Use a single global B&W toggle" → per-monitor "Show B&W toggle" checkboxes become disabled and uncheck; their state is saved as off.
   - Left-click tray → one "B&W (All Monitors)" checkbox appears at the top; per-monitor B&W toggles are gone.
   - Toggle it → all monitors go grayscale together; brightness/colour-temp still adjustable.
5. **Mode switching:** Turn global mode off in Settings → per-monitor checkboxes re-enable; user can re-tick visibility per monitor. Previously-stored per-monitor `bwFilterEnabled` values are preserved.
6. **Edge cases:** Add/remove a monitor while running (existing `RefreshMonitors` path) — confirm new monitor defaults to filter off and toggle absent unless enabled in Settings.
